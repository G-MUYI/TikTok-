# AGENTS.md - 短视频带货爆款文案生成工作流

> 本文件是项目的统一规则源，所有 AI Agent（Claude Code、Kiro、Cursor 等）都应遵循此规范。
> 执行流程和指令定义见 [viral-copywriter/SKILL.md](viral-copywriter/SKILL.md)。

## 项目概述

### 核心目标
分析爆款文案的结构、情绪和特点，生成 3-5 个适合 TikTok 短视频带货的爆款文案变体，并提供中文、英文、西班牙语和日语翻译。

### 技术栈
- **核心**：Claude Code/Kiro Skills 系统
- **存储**：本地文件系统（样本库 + 输出）
- **AI 能力**：Claude 内置的文案生成和多语言翻译能力
- **语音合成**：Fish Audio S1 情感标签集成

## 核心约束条件

### 内容质量
- **合规性**：生成的文案需符合 TikTok、抖音等平台内容政策
  - 避免敏感词、夸大宣传、违规承诺
- **风格要求**：文案风格要自然亲切，像朋友分享而非营销话术
  - ❌ 避免："限时抢购"、"错过后悔"、"史上最低"
  - ✅ 推荐："我最近发现"、"真的好用"、"分享给你们"
- **CTA（行动号召）收尾**：每段文案末尾必须包含购物车引导语
  - **不要固定话术**，需要根据文案的语境、情绪和叙事风格自然衔接
  - 核心信息点：① 引导点击左下角购物车 ② 提及当前有优惠
  - 翻译时同样要贴合目标语言的口语习惯，不要死译
  - Fish Audio 配音标签版使用 `(encouraging)` 情绪标签

### 翻译质量
- 保持种草口播语气和情绪强度（像朋友分享，不要硬广）
- 不能只是字面翻译，需要符合目标语言的表达习惯
- 可适度使用目标语言的俚语/口头禅（自然即可，避免生硬堆砌）
- 支持语言：中文、英文、西班牙语、日语
- **日语特殊约束**：日语版本必须控制在 20秒口播时长（约130-150モーラ），在不破坏爆款结构的前提下精简表达
- **日语风格建议**：短句、语气更口语；可保留拟声词/语气词；适度俚语（如「マジ」「ガチ」「やばい」「秒で」「神」等）
- **日语口吻（默认中性）**：尽量省略主语；避免明显性别化自称/语尾（如「俺/僕/あたし」「〜だぜ/〜だわ」），用中性口语表达即可

### 配音标签质量（Fish Audio）
- 情绪标签必须放在句子开头（Fish Audio 引擎规则）
- 每句话只用一个情绪标签，避免冲突
- 情绪流要匹配文案的"爆款公式"结构（钩子→展示→共鸣→暗示）
- 支持多语言配音：标签统一用英文，文案内容用目标语言

### 性能要求（目标耗时）
- **快速模式**（默认）：10秒左右
- **标准模式**：15秒左右
- **深度模式**：20-30秒左右
- **成本控制**：使用 Claude 内置能力，无额外 API 调用成本

## 样本库规范

### 自动保存
- 每次分析后自动保存到 `samples/cases/`
- 同步更新 `samples/index.json`
- 隐私保护：品牌名/产品名用占位符替换

### 样本索引格式
```json
{
  "id": "sample-NNN",
  "category": "产品类别",
  "product": "产品名",
  "date": "YYYY-MM-DD",
  "source": "来源",
  "tags": ["标签1", "标签2"],
  "emotion_score": 7,
  "key_patterns": ["模式1", "模式2"],
  "summary": "一句话摘要"
}
```

## 文件操作规范

### 路径处理（Windows 中文路径）
- **CRITICAL**：必须使用 Git Bash（MSYS2），严禁使用 PowerShell
- 路径必须用双引号包裹，必须使用正斜杠 `/`
- 支持中文路径和文件名

### 输出目录
- 生成的文案保存到 `output/` 目录
- 命名格式：`YYYYMMDD-产品名/`（统一使用中文命名，尽量避免特殊符号）
- 每个产品一个目录，内含各语言版本和 README.md

### 文件命名
- 语言文件：`zh-CN.md`、`en-US.md`、`es-ES.md`、`ja-JP.md`
- 配音文件：`fish-audio-variantN.md`
- 汇总文件：`README.md`

## AI 协作规范

### 对话原则
1. **简洁高效**：每个阶段完成后简短告知，不要冗长解释
2. **自动推进**：分析完成后自动进入下一阶段，除非用户要求暂停
3. **中文优先**：所有交互使用中文

### 用户偏好
- AI 启动时读取 `.ai/user-profile.json` 加载用户偏好
- 生成变体时优先使用用户喜欢的模式
- 避免使用用户标记为不喜欢的词汇和表达
- 用户提供反馈后更新 `user-profile.json`

### 迭代流程
1. 用户提出需求或反馈
2. AI 理解并确认需求
3. AI 执行并交付
4. 用户测试并反馈
5. 循环迭代
