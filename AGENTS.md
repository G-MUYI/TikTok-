# AGENTS.md - 短视频带货爆款文案生成工作流

> 本文件是项目的统一规则源，所有 AI Agent（Claude Code、Kiro、Cursor 等）都应遵循此规范。

## 项目概述

### 核心目标
分析爆款文案的结构、情绪和特点，生成 3-5 个适合 TikTok 短视频带货的爆款文案变体，并提供中文、英文、西班牙语和日语翻译。

### 目标用户
内容创作者

### 成功标准
- 创作效率提高
- 使用率提高

## 技术架构

### 技术栈
- **核心**：Claude Code/Kiro Skills 系统
- **存储**：本地文件系统（用于爆款文案样本库）
- **AI 能力**：Claude 内置的文案生成和多语言翻译能力
- **语音合成**：Fish Audio S1 情感标签集成（配音标签生成）

## 核心约束条件

### 性能要求
- **响应速度**：
  - 快速模式：20秒内完成（分析10秒 + 生成 + 翻译）
  - 标准模式：25秒内完成（分析15秒 + 生成 + 翻译）
  - 深度模式：40秒内完成（分析20-30秒 + 生成 + 翻译）
- **成本控制**：使用 Claude 内置能力，无额外 API 调用成本
- **分析挡位**：提供快速/标准/深度三档可选，默认快速模式

### 内容质量
- **合规性**：生成的文案需符合 TikTok、抖音等平台内容政策
  - 避免敏感词
  - 避免夸大宣传
  - 避免违规承诺
- **风格要求**：文案风格要自然亲切，像朋友分享而非营销话术
  - ❌ 避免："限时抢购"、"错过后悔"、"史上最低"
  - ✅ 推荐："我最近发现"、"真的好用"、"分享给你们"

### 翻译质量
- 保持营销语气和情绪强度
- 不能只是字面翻译
- 需要符合目标语言的表达习惯
- 支持语言：中文、英文、西班牙语、日语
- **日语特殊约束**：日语版本必须控制在 20秒口播时长（约130-150モーラ），在不破坏爆款结构的前提下精简表达

### 配音标签质量（Fish Audio 集成）🎙️
- 情绪标签必须放在句子开头（Fish Audio 引擎规则）
- 每句话只用一个情绪标签，避免冲突
- 情绪流要匹配文案的"爆款公式"结构（钩子→展示→共鸣→暗示）
- 支持多语言配音：标签统一用英文，文案内容用目标语言
- 参考文档：https://docs.fish.audio/developer-guide/best-practices/emotion-control

### 系统能力
- **可持续迭代**：系统设计支持功能扩展和优化
- **学习能力**：保存用户提供的爆款文案样本，形成知识库
- **资源留存**：样本库可持续积累，提升生成质量

## 工作流编排

### Phase 1: 样本库初始化
**目标**：建立爆款文案样本库的基础结构

**步骤**：
1. 创建 `samples/` 目录结构
2. 设计样本索引格式（JSON Schema）
3. 创建样本模板（Markdown 格式）
4. 编写样本管理文档

**交付物**：
- `samples/index.json` - 样本索引文件
- `samples/template.md` - 样本模板
- 样本管理文档

### Phase 2: Skill 核心开发
**目标**：开发爆款文案生成的核心 Skill

**步骤**：
1. 创建 `viral-copywriter/` 目录（独立 skill 包）
2. 编写 `SKILL.md` 定义文件
   - 文案分析逻辑（结构、情绪、特点提取）
   - 文案生成逻辑（3-5 个变体）
   - 多语言翻译逻辑
3. 实现样本库查询和学习机制
4. 设计用户交互流程

**交付物**：
- `viral-copywriter/SKILL.md`
- 使用说明文档

### Phase 3: 测试与优化
**目标**：验证 Skill 功能并优化体验

**步骤**：
1. 准备 5-10 个真实爆款文案样本
2. 测试文案分析准确性
3. 测试生成文案的质量和多样性
4. 测试多语言翻译质量
5. 收集反馈并迭代优化

**交付物**：
- 测试报告
- 优化后的 Skill 版本

### Phase 4: 样本库扩充
**目标**：持续积累爆款文案样本，提升生成质量

**步骤**：
1. 建立样本收集流程
2. 设计样本标注规范（标签、分类、特征）
3. 实现样本自动分析和索引
4. 定期回顾和更新样本库

**交付物**：
- 样本收集指南
- 样本标注规范
- 持续更新的样本库

## AI 协作规范

### 协作方式
**边聊边改**：用户和 AI 通过对话逐步完善功能

### 对话原则
1. **简洁明确**：每次回复聚焦当前问题，避免冗长
2. **及时反馈**：每个步骤完成后告知用户进度
3. **主动建议**：发现问题或优化点时主动提出
4. **保持上下文**：记住之前的对话内容和决策
5. **中文优先**：所有交互使用中文，Skills 指令支持中文

### 会话持久化机制
1. **启动时自动恢复**：
   - AI 启动时自动读取 `.ai/session-state.json`、`.ai/user-profile.json`、`.ai/work-history.json`
   - 如果发现上次会话未完成，主动询问是否继续
   - 根据用户偏好自动调整生成策略

2. **状态文件说明**：
   - `.ai/session-state.json` - 当前会话状态（AI自动维护）
   - `.ai/user-profile.json` - 用户偏好配置（可手动编辑）
   - `.ai/work-history.json` - 工作历史记录（AI自动维护）

3. **自动保存时机**：
   - 每次完成文案分析后
   - 每次生成变体后
   - 用户提供反馈后
   - 会话结束前

4. **恢复指令**：
   - `/恢复会话` - 恢复上次未完成的工作
   - `/查看历史` - 查看最近的工作历史
   - `/清空状态` - 清空当前会话状态，重新开始

5. **隐私保护**：
   - 状态文件仅保存在本地
   - 不包含敏感的文案内容（仅保存引用）
   - 用户可随时删除状态文件

### Skills 指令系统
1. **中文指令优先**：
   - 主要使用中文指令，如 `/爆款文案`、`/初始化`
   - 支持英文指令作为备选，如 `/vc`、`/init`
   - 详细指令列表见 [.ai/SKILLS_INDEX.md](.ai/SKILLS_INDEX.md)

2. **指令格式**：
   ```
   /爆款文案                          # 基础指令
   /爆款文案 --快速                   # 带模式参数
   /爆款文案 --变体数量=5 --风格=专业  # 带多个参数
   ```

3. **全局命令**：
   - `/帮助` 或 `/help` - 显示帮助
   - `/技能列表` 或 `/skills` - 列出所有 Skills
   - `/配置` 或 `/config` - 查看配置
   - `/配音标签` 或 `/fish-audio` - 生成 Fish Audio 配音标签

4. **触发优先级**：
   - 显式指令 > 自动识别
   - 上下文感知，避免意外切换 Skills

### 文件操作规范
1. **生命周期管理**：
   - 生成新文件后，检查是否有被替代的旧文件
   - 列出冗余文件清单，建议用户删除
   - 经确认后执行清理
2. **路径处理**（Windows 中文路径）：
   - **CRITICAL**：必须使用 Git Bash（MSYS2），严禁使用 PowerShell
   - 所有涉及中文路径的操作必须通过 Bash 工具执行
   - 路径必须用双引号包裹
   - 必须使用正斜杠 `/`，不能使用反斜杠 `\`
   - 创建中文文件夹示例：`mkdir -p "samples/案例库/美妆类"`
   - 文件操作示例：`touch "output/生成文案_20260304.md"`

### 迭代流程
1. 用户提出需求或反馈
2. AI 理解并确认需求
3. AI 提出实现方案
4. 用户确认或调整
5. AI 执行并交付
6. 用户测试并反馈
7. 循环迭代

## 样本库规范

### 样本格式
每个爆款文案样本应包含：

```markdown
# 样本标题

## 原文
[原始文案内容]

## 分析
- **结构**：[开头/中间/结尾的结构特点]
- **情绪**：[情绪类型和强度]
- **特点**：[独特的表达方式、修辞手法等]
- **目标受众**：[适合的人群]
- **产品类型**：[适合的产品类别]

## 标签
#标签1 #标签2 #标签3

## 元数据
- 收集日期：YYYY-MM-DD
- 来源平台：TikTok/抖音/其他
- 数据表现：[点赞/评论/转化等数据，如有]
```

### 样本索引格式
```json
{
  "version": "1.0.0",
  "last_updated": "2026-03-04",
  "total_samples": 0,
  "samples": [
    {
      "id": "001",

      "file": "cases/001-xxx.md",
      "tags": ["标签1", "标签2"],
      "product_type": "产品类型",
      "emotion": "情绪类型",
      "added_date": "2026-03-04"
    }
  ]
}
```
