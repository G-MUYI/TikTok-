# 测试文件说明

## 📁 目录结构

```
tests/
├── README.md                        # 本文件
├── SESSION_PERSISTENCE_TEST.md      # 会话持久化测试指南
├── test-session.sh                  # 快速测试脚本
└── fixtures/                        # 测试文案
    ├── test-beauty.txt              # 美妆类测试文案
    ├── test-digital.txt             # 数码类测试文案
    └── test-food.txt                # 食品类测试文案
```

## 🚀 快速开始

### 1. 运行测试脚本

```bash
cd "d:/AI导航网站及资源网站/AI编程小工具/TikTok-爆款文案生成器/TikTok-"
./tests/test-session.sh
```

这个脚本会：
- 检查状态文件是否存在
- 显示状态文件内容
- 检查文件修改时间
- 检查测试文案

### 2. 使用测试文案进行实际测试

#### 测试场景1：基础功能测试

```
/爆款文案
[复制 tests/fixtures/test-beauty.txt 的内容]
```

等待AI完成分析和生成变体，然后关闭窗口。

#### 测试场景2：会话恢复测试

重新打开窗口，观察AI是否自动提示恢复会话。

如果没有自动提示，输入：
```
/恢复会话
```

#### 测试场景3：用户偏好学习测试

完成一次完整流程后，告诉AI你的偏好：
```
我最喜欢变体2，因为更口语化
```

然后使用另一个测试文案，观察AI是否应用了你的偏好。

## 🧪 测试文案说明

### test-beauty.txt - 美妆类
- **产品类型**：面膜
- **情绪基调**：兴奋、惊喜
- **特点**：用户见证、效果对比
- **适合测试**：情绪分析、认知劫持原理（社会认同、用户见证）
- **预期变体风格**：故事叙述、好奇心驱动、效果对比

### test-digital.txt - 数码类
- **产品类型**：蓝牙耳机
- **情绪基调**：满意、推荐
- **特点**：功能介绍、使用体验
- **适合测试**：结构分析、多维度价值呈现
- **预期变体风格**：功能罗列、场景化描述、对比分析

### test-food.txt - 食品类
- **产品类型**：零食
- **情绪基调**：安利、分享
- **特点**：口味描述、社交传播
- **适合测试**：语言特点、社交传播机制
- **预期变体风格**：朋友安利、口味描述、稀缺性驱动

## 🎯 测试用例示例

### 用例1：基础分析测试
```
/爆款文案
[复制 test-beauty.txt 内容]
```
**预期结果**：
- 完整的深度分析（结构、情绪、语言、心理学原理）
- 自动保存到样本库
- 生成3-5个变体

### 用例2：参数化测试
```
/爆款文案 --变体数量=5 --风格=专业
[复制 test-digital.txt 内容]
```
**预期结果**：
- 生成5个变体（而不是默认的3-5个）
- 变体风格偏向专业（而不是友好）

### 用例3：快速模式测试
```
/爆款文案 --快速 --跳过保存
[复制 test-food.txt 内容]
```
**预期结果**：
- 跳过详细分析
- 直接生成变体
- 不保存到样本库

### 用例4：仅分析模式测试
```
/爆款文案 --仅分析
[复制 test-beauty.txt 内容]
```
**预期结果**：
- 只进行深度分析
- 不生成变体
- 保存分析结果到样本库

### 用例5：翻译测试
```
/爆款文案
[复制 test-digital.txt 内容]
[等待生成变体]
翻译变体2
```
**预期结果**：
- 提供中文、英文、西班牙语、日语版本
- 翻译符合文化习惯
- 保持情绪强度

### 用例6：反馈学习测试
```
/爆款文案
[复制 test-beauty.txt 内容]
[等待生成变体]
我最喜欢变体2，因为更口语化，以后多生成这种风格
```
**预期结果**：
- AI 记录用户偏好
- 更新 user-profile.json
- 下次生成时应用偏好

### 用例7：批量处理测试
```
/爆款文案 --批量
文案1：[test-beauty.txt 内容]

文案2：[test-food.txt 内容]

文案3：[test-digital.txt 内容]
```
**预期结果**：
- AI 询问处理方式（逐个/批量）
- 按选择的方式处理所有文案
