# 测试文件说明

## 目录结构

```
tests/
├── README.md                        # 本文件
└── fixtures/                        # 测试文案
    ├── test-beauty.txt              # 美妆类测试文案
    ├── test-digital.txt             # 数码类测试文案
    └── test-food.txt                # 食品类测试文案
```

## 测试文案说明

### test-beauty.txt - 美妆类
- **产品类型**：面膜
- **情绪基调**：兴奋、惊喜
- **适合测试**：情绪分析、认知劫持原理

### test-digital.txt - 数码类
- **产品类型**：蓝牙耳机
- **情绪基调**：满意、推荐
- **适合测试**：结构分析、多维度价值呈现

### test-food.txt - 食品类
- **产品类型**：零食
- **情绪基调**：安利、分享
- **适合测试**：语言特点、社交传播机制

## 测试用例

### 用例1：基础分析
```
/爆款文案
[复制 test-beauty.txt 内容]
```
**预期**：完整分析 + 自动保存样本 + 3-5个变体

### 用例2：参数化
```
/爆款文案 --变体数量=5 --风格=专业
[复制 test-digital.txt 内容]
```
**预期**：5个专业风格变体

### 用例3：快速模式
```
/爆款文案 --快速 --跳过保存
[复制 test-food.txt 内容]
```
**预期**：快速分析 + 变体，不保存样本

### 用例4：仅分析
```
/爆款文案 --仅分析
[复制 test-beauty.txt 内容]
```
**预期**：只分析不生成变体

### 用例5：翻译
```
/爆款文案
[等待复刻版（推荐）与四语翻译输出]
```
**预期**：默认对推荐版输出中/英/西/日四语翻译，日语控制在 20秒

### 用例6：反馈学习
```
我最喜欢变体2，因为更口语化
```
**预期**：更新 user-profile.json，下次应用偏好

### 用例7：批量处理
```
/爆款文案 --批量
文案1：[test-beauty.txt]
文案2：[test-food.txt]
```
**预期**：询问处理方式后执行
