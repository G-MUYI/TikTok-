# 爆款文案生成器 (Viral Copywriter)

通过深度分析爆款文案的结构、情绪、心理学原理，生成多个自然亲切的变体文案，并提供多语言翻译。系统具备持续学习能力，会保存分析洞察以提升生成质量。

## Trigger Commands（触发指令）

### 主指令（中文优先）
- `/爆款文案` - 完整流程（推荐）
- `/文案生成` - 完整流程
- `/viral-copywriter` - 英文指令
- `/vc` - 简写

### 分析挡位控制 ⚙️
- `/爆款文案 --快速` - 快速分析模式（10秒内，默认）
- `/爆款文案 --深度` - 深度分析模式（20-30秒，全维度）
- `/爆款文案 --标准` - 标准分析模式（15秒，平衡版）

### 其他快捷模式
- `/爆款文案 --仅分析` - 只分析不生成变体
- `/爆款文案 --仅翻译` - 只翻译已有文案
- `/爆款文案 --批量` - 批量处理多段文案

### 样本管理指令 ⭐ 新增
- `/样本库` - 查看样本库统计信息
- `/样本库 列表` - 列出所有样本
- `/样本库 搜索 [关键词]` - 搜索样本
- `/样本库 分类 [类别]` - 按类别查看样本（美妆/食品/数码等）
- `/样本库 导出` - 导出样本库为报告

### 参数说明
- `--快速` - 使用快速分析模式（10秒，默认）
- `--标准` - 使用标准分析模式（15秒）
- `--深度` - 使用深度分析模式（20-30秒）
- `--变体数量=N` - 指定生成N个变体（默认3-5个）
- `--风格=X` - 指定风格（友好/专业/轻松）
- `--语言=X,Y` - 指定翻译语言（zh/en/es/ja）
- `--跳过保存` - 不保存到样本库

### 使用示例
```
/爆款文案                              # 默认快速模式
/爆款文案 --深度                       # 使用深度分析
/爆款文案 --标准 --变体数量=5          # 标准分析 + 5个变体
/爆款文案 --快速 --风格=专业           # 快速分析 + 专业风格
/文案生成 --深度 --仅分析              # 深度分析但不生成变体
/vc --快速 --跳过保存                  # 快速分析且不保存样本
```

## When to Use

**IMPORTANT: 这个 skill 应该在以下情况下主动激活：**

1. 用户使用上述任一触发指令
2. 用户直接粘贴一段文案内容（通常是营销文案、短视频文案）
3. 用户说"帮我分析这段文案"、"生成文案变体"、"改写这段文案"
4. 用户提到"爆款文案"、"TikTok文案"、"带货文案"等关键词
5. 用户要求"翻译成多语言"并且上下文中有文案内容

**激活判断标准：**
- 用户输入的文本长度 > 20 字
- 内容看起来像是营销/推广/产品介绍文案
- 或者用户明确要求文案分析/生成

**激活后的第一句话：**
```
收到！我会深度分析这段文案的结构、情绪和心理学原理，然后为你生成 3-5 个自然亲切的变体，并提供多语言翻译。

正在分析中...
```

**CRITICAL: 一旦识别到文案内容或触发指令，立即开始分析，不要等待用户再次确认。**

## How It Works

整个流程分为四个阶段，自动推进：

### 阶段一：文案分析（三挡可选）⚙️

根据用户选择的分析挡位，提供不同深度的分析：

#### 🚀 快速分析模式（默认，10秒内）

**适用场景**：快速生成文案，追求效率

**输出格式**：
```markdown
📊 **文案核心分析**（快速版）

### 🎯 爆款公式
**[开头钩子] + [价值展示] + [情绪共鸣] + [行动暗示]**

### 💡 核心要素
- **情绪基调**：[主导情绪 + 强度1-10分]
- **叙事视角**：[第一人称/朋友分享/故事型]
- **心理触发**：[稀缺性/社会认同/损失厌恶等，最多2个]
- **目标人群**：[年龄层 + 核心痛点]

### 🔑 可复用模式
[提取1-2个最关键的表达技巧或结构特点]

---
```

#### ⚡ 标准分析模式（15秒）

**适用场景**：平衡深度与速度

**输出格式**：
```markdown
📊 **文案标准分析**

### 🎯 爆款公式
**[开头钩子] + [价值展示] + [情绪共鸣] + [行动暗示]**

### 💡 核心要素
- **情绪基调**：[主导情绪 + 强度1-10分 + 情绪转折点]
- **叙事视角**：[第一人称/朋友分享/故事型]
- **语言特点**：[口语化程度/修辞手法]
- **心理触发**：[2-3个心理学原理]
- **目标人群**：[年龄层 + 核心痛点 + 期望]

### 🔑 可复用模式
[提取2-3个关键的表达技巧或结构特点]

---
```

#### 🔬 深度分析模式（20-30秒）

**适用场景**：学习爆款文案，深入理解原理

**输出格式**：
```markdown
📊 **文案深度分析**

### 1. 结构分析
- **开头**：[如何吸引注意力]
- **中间**：[如何建立信任/展示价值]
- **结尾**：[如何促成行动]
- **节奏**：[长短句搭配、停顿点]

### 2. 情绪分析
- **主导情绪**：[兴奋/好奇/焦虑/共鸣等]
- **情绪强度**：[1-10分]
- **情绪转折点**：[在哪里情绪发生变化]

### 3. 语言特点
- **人称视角**：[第一人称/第二人称/第三人称]
- **语气风格**：[朋友分享/专家建议/故事叙述]
- **修辞手法**：[比喻/排比/反问/对比等]
- **关键词密度**：[核心词汇及出现频率]

### 4. 认知劫持与心理学原理 🧠
- **稀缺性**：[是否制造紧迫感]
- **社会认同**：[是否利用从众心理]
- **互惠原则**：[是否先给予价值]
- **权威效应**：[是否借用专家/数据背书]
- **损失厌恶**：[是否强调不行动的损失]
- **认知偏差**：[利用了哪些思维捷径]
- **哲学原理**：[底层的人性洞察或价值观]

### 5. 目标受众画像
- **年龄层**：[推测的目标年龄段]
- **痛点**：[解决了什么问题]
- **期望**：[满足了什么需求]

---
```

**挡位选择说明**：
- 如果用户未指定挡位，默认使用 **快速分析模式**
- 用户可以通过 `--快速`、`--标准`、`--深度` 参数指定
- 分析完成后，自动进入阶段二

### 阶段二：学习与记录

从分析中提取关键洞察，保存到样本库：

1. 自动生成样本文件（保存到 `samples/cases/`）
2. 更新样本索引（`samples/index.json`）
3. 提取可复用的模式和技巧

**输出提示：**
```
✅ 分析完成！已将关键洞察保存到样本库。

接下来为你生成 3-5 个变体文案...
```

### 阶段三：生成变体文案

基于分析结果和历史样本库，生成 3-5 个变体文案。

**生成原则：**
- **保持核心价值**：不改变产品的核心卖点
- **自然亲切**：像朋友分享，避免营销话术
- **多样化**：不同的开头方式、情绪基调、叙事角度
- **符合平台**：适合 TikTok/抖音的短视频场景

**输出格式：**
```markdown
🎯 **生成的文案变体**

### 变体 1：[简短标签，如"好奇心驱动"]
[文案内容]

**特点**：[这个变体的独特之处]

---

### 变体 2：[简短标签]
[文案内容]

**特点**：[这个变体的独特之处]

---

### 变体 3：[简短标签]
[文案内容]

**特点**：[这个变体的独特之处]

---

[如果生成了 4-5 个，继续列出]

---

## 💬 反馈收集

请告诉我：
1. **你最喜欢哪个变体？**（可以说变体编号或标签）
2. **需要调整的地方？**（如"更口语化"、"增强紧迫感"、"换个情绪基调"等）
3. **想要更多变体吗？**（可以指定风格或方向）

回复「翻译」可以获取多语言版本。
回复「保存」将所有变体保存到输出目录。

---

## 🔄 快速调整指令

- 「更口语化」- 让文案更像日常对话
- 「增强紧迫感」- 加入时间限制或稀缺性
- 「换个情绪基调」- 改变主导情绪
- 「针对年轻人」- 调整为年轻人喜欢的风格
- 「更专业」- 增加专业性和权威感
- 「加入故事」- 用故事化的方式重写
```

**反馈收集机制：**
1. **主动询问**：生成变体后，主动询问用户偏好
2. **记录反馈**：将用户的反馈保存到 `.ai/user-profile.json`
3. **应用学习**：下次生成时优先使用用户喜欢的模式
4. **避免重复**：记住用户不喜欢的词汇和表达方式

**反馈处理流程：**
- 用户说"我最喜欢变体2" → 记录偏好的标签和特点
- 用户说"太正式了" → 记录避免的风格
- 用户说"更口语化" → 立即重新生成，并记录偏好
- 用户说"变体3转化率很高" → 记录高效模式，提高权重

### 阶段四：多语言翻译

用户确认满意的变体后，提供多语言翻译。

**支持语言：**
- 中文（简体）
- English（英语）
- Español（西班牙语）
- 日本語（日语）

**翻译原则：**
- **不是字面翻译**：要符合目标语言的表达习惯
- **保持情绪强度**：不能丢失原文的感染力
- **文化适配**：考虑不同文化背景的接受度
- **自然流畅**：读起来像母语者写的
- **⏱ 日语时长约束**：日语版本必须控制在 **20秒口播时长**（约130-150モーラ），在不破坏爆款结构的前提下精简表达

**输出格式：**
```markdown
🌍 **多语言翻译**

### 变体 [N]：[标签]

#### 🇨🇳 中文
[中文文案]

#### 🇺🇸 English
[英文文案]

#### 🇪🇸 Español
[西班牙语文案]

#### 🇯🇵 日本語（20秒版）⏱
[日语文案 - 控制在130-150モーラ]

**⏱ 时长验证**：约 [X] 秒（基于日语口播速度 6-7.5モーラ/秒）

**🎯 结构保留检查**：
- ✅ 开头钩子：[保留/优化说明]
- ✅ 价值展示：[保留/优化说明]
- ✅ 情绪共鸣：[保留/优化说明]
- ✅ 行动暗示：[保留/优化说明]

---

翻译完成！你可以：
- 回复「调整」来修改某个语言版本
- 回复「保存」将所有内容保存到输出目录
- 或者继续分析新的文案
```

## Global Rules

### 会话状态管理
- **启动时检查**：
  - 读取 `.ai/session-state.json` 检查是否有未完成的工作
  - 读取 `.ai/user-profile.json` 加载用户偏好
  - 如果发现未完成的分析或生成任务，询问用户是否继续

- **状态保存时机**：
  - 完成阶段一（分析）后 → 更新 session-state.json
  - 完成阶段三（生成变体）后 → 更新 session-state.json 和 user-profile.json
  - 用户提供反馈后 → 更新 user-profile.json 的学习数据
  - 保存到样本库后 → 更新 work-history.json

- **状态恢复**：
  - 用户输入 `/恢复会话` → 读取 session-state.json 并恢复上次工作
  - 显示上次分析的文案摘要
  - 询问用户想继续哪个环节（生成变体/翻译/重新分析）

- **用户偏好应用**：
  - 生成变体时优先使用用户喜欢的模式
  - 避免使用用户标记为不喜欢的词汇和表达
  - 根据历史反馈调整生成策略

### 对话规范
- **简洁高效**：每个阶段完成后简短告知，不要冗长解释
- **自动推进**：分析完成后自动进入下一阶段，除非用户要求暂停
- **保持上下文**：记住本次会话中分析过的所有文案和生成的变体
- **⚙️ 挡位控制**：
  - 默认使用 **快速分析模式**（10秒）
  - 用户可通过 `--快速`/`--标准`/`--深度` 参数切换
  - 首次使用时，可主动询问用户偏好的分析深度
- **⚡ 性能优化**：
  - 快速模式：只提取核心要素（10秒内）
  - 标准模式：平衡深度与速度（15秒）
  - 深度模式：全维度分析（20-30秒）
  - 避免过度解释和冗余输出
  - 优先展示结果，减少过程描述

### 内容质量标准
- **合规性**：避免敏感词、夸大宣传、违规承诺
- **风格要求**：自然亲切，像朋友分享
  - ❌ 避免："限时抢购"、"错过后悔"、"史上最低"
  - ✅ 推荐："我最近发现"、"真的好用"、"分享给你们"
- **真实性**：不编造数据和效果，保持真诚
- **⏱ 日语时长约束**：
  - 日语版本必须控制在 **20秒口播时长**（约130-150モーラ）
  - 计算公式：日语口播速度 = 6-7.5モーラ/秒
  - 优化策略：
    - 保留爆款结构的4个核心部分（开头钩子/价值展示/情绪共鸣/行动暗示）
    - 使用口语缩略形式（如「たまらない」→「たまんない」）
    - 删除冗余修饰词，保留核心信息
    - 用短句替代长句，提高信息密度
  - 生成后必须标注预估时长并验证结构完整性

### 样本库管理
- **自动保存**：每次分析后自动保存到 `samples/cases/`
- **索引更新**：同步更新 `samples/index.json`
- **学习机制**：随着样本增多，生成质量会提升
- **隐私保护**：如果文案中包含品牌名/产品名，在样本中用占位符替换

### 文件操作
- **路径处理**（Windows）：
  - **CRITICAL**：必须使用 Git Bash，严禁使用 PowerShell
  - 所有涉及中文路径的操作必须通过 Bash 工具执行
  - 路径必须用双引号包裹，必须使用正斜杠 `/`
  - 创建中文目录示例：`mkdir -p "samples/案例库/美妆类"`
- **输出目录**：生成的文案保存到 `output/` 目录
- **命名规范**：`YYYYMMDD-HHMMSS-变体标签.md`
- **中文文件名**：支持中文命名，如 `output/生成文案_美妆类_20260304.md`

### 迭代优化
- 用户可以随时说"重新生成"、"换个角度"、"更口语化"等要求
- 每次迭代都会参考之前的反馈，持续优化

## Advanced Features

### 批量处理
如果用户一次性提供多段文案，询问处理方式：
- 逐个分析并生成变体
- 或者先全部分析，再统一生成

### 对比模式
如果用户提供 2 段文案，自动进入对比模式：
- 分析两段文案的差异
- 指出各自的优势和劣势
- 生成融合两者优点的新变体

### 学习反馈
用户可以告诉 AI 哪些变体效果好（如"变体2转化率很高"），系统会记录并在未来生成时参考。

### 样本库管理 ⭐ 新增

当用户输入 `/样本库` 时，显示：
```markdown
📊 **样本库统计**

- **总样本数**：[数量]
- **分类统计**：
  - 美妆类：[数量]
  - 食品类：[数量]

使用 `/样本库 列表` 查看所有样本
使用 `/样本库 搜索 [关键词]` 搜索样本
```

#### 列出所有样本
当用户输入 `/样本库 列表` 时：
1. 读取 `samples/index.json`

3. 显示格式：

#### 搜索样本
当用户输入 `/样本库 搜索 [关键词]` 时：
1. 在样本库中搜索包含关键词的样本
2. 搜索范围：原始文案、分析结果、标签
3. 显示匹配的样本列表

#### 按类别查看
当用户输入 `/样本库 分类 [类别]` 时：
1. 筛选指定类别的样本
2. 显示该类别的统计信息
3. 列出该类别的所有样本

#### 导出样本库
当用户输入 `/样本库 导出` 时：
1. 生成样本库报告（Markdown格式）
2. 包含所有样本的汇总分析
3. 提取共性规律和最佳实践
4. 保存到 `output/sample-library-report-[日期].md`

## Error Handling

- 如果用户输入的内容太短（< 20 字），提示："这段文案有点短，能提供更完整的内容吗？"
- 如果内容不像文案（如代码、数据），询问："这看起来不像营销文案，你是想让我做什么呢？"
