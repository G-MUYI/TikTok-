# 爆款文案生成器 (Viral Copywriter)

> 质量标准和约束条件见 [AGENTS.md](../AGENTS.md)

## Trigger Commands

### 主指令
- `/爆款文案` - 完整流程（推荐）
- `/文案生成` - 完整流程
- `/viral-copywriter` / `/vc` - 英文指令

### 分析挡位
- `/爆款文案 --快速` - 快速分析（10秒，默认）
- `/爆款文案 --标准` - 标准分析（15秒）
- `/爆款文案 --深度` - 深度分析（20-30秒）

### 快捷模式
- `--仅分析` - 只分析不生成变体
- `--仅翻译` - 只翻译已有文案
- `--配音` - 完整流程 + Fish Audio 配音标签
- `--批量` - 批量处理多段文案

### 配音标签指令
- `/配音标签` / `/fish-audio` - 生成 Fish Audio 情感标签版本
- `--语言=zh` - 只生成指定语言的配音版
- `--情绪=激动` - 指定整体情绪基调

### 样本管理指令
- `/样本库` - 查看样本库统计
- `/样本库 列表` - 列出所有样本
- `/样本库 搜索 [关键词]` - 搜索样本
- `/样本库 分类 [类别]` - 按类别查看
- `/样本库 导出` - 导出样本库报告

### 参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--变体数量=N` | 生成N个版本（含 1 个复刻版/推荐版） | 3 |
| `--风格=X` | 友好/专业/轻松 | friendly |
| `--语言=X,Y` | 翻译语言 | zh,en,es,ja |
| `--日语口吻=X` | neutral/feminine/masculine | neutral |
| `--跳过翻译` | 跳过多语言翻译阶段 | false |
| `--翻译全部` | 翻译所有变体（默认只翻译推荐版） | false |
| `--跳过保存` | 不保存到样本库 | false |
| `--配音` | 生成配音标签 | false |
| `--情绪=X` | 配音情绪基调 | auto |

## When to Use

**IMPORTANT: 这个 skill 应该在以下情况下主动激活：**

1. 用户使用上述任一触发指令
2. 用户直接粘贴一段文案内容（长度 > 20 字，内容像营销文案）
3. 用户说"帮我分析这段文案"、"生成文案变体"、"改写这段文案"
4. 用户提到"爆款文案"、"TikTok文案"、"带货文案"等关键词

**CRITICAL: 一旦识别到文案内容或触发指令，立即开始分析，不要等待用户再次确认。**

**激活后的第一句话：**
```
收到！正在分析中...
```

## How It Works

整个流程分为四个阶段，**关键节点询问用户**：

### 阶段一：文案分析（三挡可选）

**重点：认知劫持分析**

根据挡位提供不同深度的分析，**必须包含认知劫持维度**：

#### 快速分析（默认）
输出：爆款公式 + 核心要素（情绪/视角/心理触发/目标人群）+ **认知劫持手法** + 可复用模式
参考模板：`templates/analysis-output.md`（快速版仅输出核心部分）

#### 标准分析
在快速版基础上增加：语言特点、情绪转折点、**认知偏见利用**

#### 深度分析
全维度：结构分析 + 情绪分析 + 语言特点 + **认知劫持与心理学原理（深度拆解）** + 目标受众画像
参考模板：`templates/analysis-output.md`（完整版）

**认知劫持分析必须包含：**
- 使用的认知偏见（锚定效应/损失厌恶/社会认同/稀缺性等）
- 注意力劫持手法（开头钩子/反问/反差/数字冲击等）
- 决策路径设计（如何引导用户从好奇→信任→行动）

### 阶段二：学习与记录

1. 自动生成样本文件（保存到 `samples/cases/`）
2. 更新样本索引（`samples/index.json`）
3. 提取可复用的模式和技巧

参考模板：`templates/sample-save.md`

输出提示：`✅ 分析完成！已将关键洞察保存到样本库。`

**自动学习迭代（必须执行）：**
- 每次生成前读取 `.ai/user-profile.json`（避免词、偏好风格、喜欢的模式），把偏好应用到后续“复刻版/推荐版”里
- 每次分析后把样本写入 `samples/`，形成可复用的“爆款模式库”
- 用户反馈后更新 `.ai/user-profile.json`（喜欢/不喜欢的表达、偏好模式权重），下次优先采用

### 阶段三：生成变体文案（从认知劫持角度）

基于分析结果和历史样本库，生成 **3-5 个变体**（总数由 `--变体数量` 决定）。

**核心要求：每个变体使用不同的认知劫持策略**

**执行要点：**
- 先读取 `.ai/user-profile.json` 的 `avoid_words / favorite_patterns / preferred_tone`（如果文件存在）
- 再参考 `samples/index.json` 中同类样本的 `key_patterns`
- **每个变体必须标注使用的认知劫持手法**（如：锚定效应、损失厌恶、社会认同等）

**变体生成策略（每个变体不同角度）：**
1. **复刻版**：高度还原原文的认知劫持结构和节奏
2. **好奇心驱动**：利用信息缺口、反问、悬念制造注意力劫持
3. **损失厌恶**：强调不行动的损失、错过的代价
4. **社会认同**：利用从众心理、用户见证、流行趋势
5. **反差对比**：前后反差、问题-解决方案对比

**所有变体共同原则：**
- **种草型口播**：短句/停顿自然，像视频口播；多用第一人称体验与细节
- **朋友分享感**：克制营销话术，不要”硬推销”；允许适度俚语/口头禅（自然即可）
- **保留平台习惯 CTA**：必须有购物车引导，但要贴合语境（不要固定话术）

参考模板：`templates/variant-output.md`

**生成后立即询问：**
```
已生成 N 个变体，每个使用不同的认知劫持策略。
你最喜欢哪个？（回复变体编号，如：变体2）
```

**反馈收集：**
用户选择后，将反馈保存到 `.ai/user-profile.json`：
- 记录偏好的认知劫持策略和风格特点
- 提高该策略在未来生成中的权重

### 阶段四：多语言翻译（询问用户）

**用户选择变体后，询问翻译需求：**
```
需要翻译成哪些语言？
1. 中文+英文+西班牙语+日语（全部）
2. 仅英文
3. 仅日语
4. 自定义（请指定语言）
5. 不需要翻译
```

根据用户选择进行翻译。

**翻译原则：**
- 文化适配，非字面翻译
- 保持情绪强度和感染力
- 保持“朋友分享”的口语口播感（可适度使用俚语/口头禅，但不要过度）
- 日语版本控制在 20秒口播时长（约130-150モーラ）
- **日语特殊特性保留**：短句、感叹/反问、拟声词/语气词、轻度俚语（如「マジ」「ガチ」「やばい」「秒で」「神」等，适量即可）
- **日语口吻默认中性**：尽量省略主语；避免明显性别化自称/语尾（如「俺/僕/あたし」「〜だぜ/〜だわ」），用中性口语 + 轻度俚语即可
- 日语生成后必须标注预估时长并验证结构完整性（钩子→展示→共鸣→行动暗示）

参考模板：`templates/translation-output.md`

### 阶段五：Fish Audio 配音标签生成（可选）

翻译完成后询问：
```
需要生成 Fish Audio 配音标签吗？（回复：需要 / 不需要）
```

用户确认后生成配音标签。

**情绪-文案结构映射表：**

| 文案结构 | 推荐情绪标签 |
|----------|-------------|
| 开头钩子（好奇型） | `(curious)` `(surprised)` |
| 开头钩子（痛点型） | `(empathetic)` `(concerned)` |
| 价值展示 | `(excited)` `(confident)` `(impressed)` |
| 功能实证 | `(confident)` `(enthusiastic)` |
| 情绪共鸣 | `(happy)` `(friendly)` `(warm)` |
| 行动暗示 | `(encouraging)` `(excited)` |
| 反差/对比 | `(skeptical)` → `(impressed)` |

**变体风格-情绪流：**

| 变体风格 | 情绪流 |
|----------|--------|
| 好奇心驱动 | curious → excited → impressed → excited |
| 故事叙述 | concerned → relieved → excited → happy |
| 用户见证 | friendly → satisfied → impressed → encouraging |
| 反差对比 | skeptical → curious → impressed → confident |
| 痛点解决 | empathetic → confident → excited → encouraging |

参考模板：`templates/fish-audio-output.md`

## Advanced Features

### 批量处理
用户一次性提供多段文案时，询问：逐个分析 or 先全部分析再统一生成

### 对比模式
用户提供 2 段文案时，自动对比分析优劣，生成融合优点的新变体

### 样本库管理
- `/样本库` → 显示统计（总数/分类/最近添加）
- `/样本库 列表` → 按时间倒序列出，显示摘要/标签/情绪强度
- `/样本库 搜索 [关键词]` → 在文案/分析/标签中搜索
- `/样本库 分类 [类别]` → 筛选指定类别
- `/样本库 导出` → 生成汇总报告到 output/

## Error Handling

- 输入太短（< 20 字）→ "这段文案有点短，能提供更完整的内容吗？"
- 内容不像文案 → "这看起来不像营销文案，你是想让我做什么呢？"
- 翻译请求不明确 → 列出可选语言让用户选择
