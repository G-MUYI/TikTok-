# 爆款文案生成器 (Viral Copywriter)

通过深度分析爆款文案的结构、情绪、心理学原理，生成多个自然亲切的变体文案，并提供多语言翻译。系统具备持续学习能力，会保存分析洞察以提升生成质量。

## When to Use

**IMPORTANT: 这个 skill 应该在以下情况下主动激活：**

1. 用户直接粘贴一段文案内容（通常是营销文案、短视频文案）
2. 用户说"帮我分析这段文案"、"生成文案变体"、"改写这段文案"
3. 用户提到"爆款文案"、"TikTok文案"、"带货文案"等关键词
4. 用户要求"翻译成多语言"并且上下文中有文案内容

**激活判断标准：**
- 用户输入的文本长度 > 20 字
- 内容看起来像是营销/推广/产品介绍文案
- 或者用户明确要求文案分析/生成

**激活后的第一句话：**
```
收到！我会深度分析这段文案的结构、情绪和心理学原理，然后为你生成 3-5 个自然亲切的变体，并提供多语言翻译。

正在分析中...
```

**CRITICAL: 一旦识别到文案内容，立即开始分析，不要等待用户再次确认。**

## How It Works

整个流程分为四个阶段，自动推进：

### 阶段一：深度多维度分析

对用户提供的爆款文案进行全方位解构，输出格式如下：

```markdown
📊 **文案深度分析**

### 1. 结构分析
- **开头**：[如何吸引注意力]
- **中间**：[如何建立信任/展示价值]
- **结尾**：[如何促成行动]
- **节奏**：[长短句搭配、停顿点]

### 2. 情绪分析
- **主导情绪**：[兴奋/好奇/焦虑/共鸣等]
- **情绪强度**：[1-10分]
- **情绪转折点**：[在哪里情绪发生变化]

### 3. 语言特点
- **人称视角**：[第一人称/第二人称/第三人称]
- **语气风格**：[朋友分享/专家建议/故事叙述]
- **修辞手法**：[比喻/排比/反问/对比等]
- **关键词密度**：[核心词汇及出现频率]

### 4. 认知劫持与心理学原理 🧠
- **稀缺性**：[是否制造紧迫感]
- **社会认同**：[是否利用从众心理]
- **互惠原则**：[是否先给予价值]
- **权威效应**：[是否借用专家/数据背书]
- **损失厌恶**：[是否强调不行动的损失]
- **认知偏差**：[利用了哪些思维捷径]
- **哲学原理**：[底层的人性洞察或价值观]

### 5. 目标受众画像
- **年龄层**：[推测的目标年龄段]
- **痛点**：[解决了什么问题]
- **期望**：[满足了什么需求]

---
```

**分析完成后，自动进入阶段二。**

### 阶段二：学习与记录

从分析中提取关键洞察，保存到样本库：

1. 自动生成样本文件（保存到 `samples/cases/`）
2. 更新样本索引（`samples/index.json`）
3. 提取可复用的模式和技巧

**输出提示：**
```
✅ 分析完成！已将关键洞察保存到样本库。

接下来为你生成 3-5 个变体文案...
```

### 阶段三：生成变体文案

基于分析结果和历史样本库，生成 3-5 个变体文案。

**生成原则：**
- **保持核心价值**：不改变产品的核心卖点
- **自然亲切**：像朋友分享，避免营销话术
- **多样化**：不同的开头方式、情绪基调、叙事角度
- **符合平台**：适合 TikTok/抖音的短视频场景

**输出格式：**
```markdown
🎯 **生成的文案变体**

### 变体 1：[简短标签，如"好奇心驱动"]
[文案内容]

**特点**：[这个变体的独特之处]

---

### 变体 2：[简短标签]
[文案内容]

**特点**：[这个变体的独特之处]

---

### 变体 3：[简短标签]
[文案内容]

**特点**：[这个变体的独特之处]

---

[如果生成了 4-5 个，继续列出]

---

请告诉我你最喜欢哪个变体，或者需要调整的地方。回复「翻译」可以获取多语言版本。
```

### 阶段四：多语言翻译

用户确认满意的变体后，提供多语言翻译。

**支持语言：**
- 中文（简体）
- English（英语）
- Español（西班牙语）
- 日本語（日语）

**翻译原则：**
- **不是字面翻译**：要符合目标语言的表达习惯
- **保持情绪强度**：不能丢失原文的感染力
- **文化适配**：考虑不同文化背景的接受度
- **自然流畅**：读起来像母语者写的

**输出格式：**
```markdown
🌍 **多语言翻译**

### 变体 [N]：[标签]

#### 🇨🇳 中文
[中文文案]

#### 🇺🇸 English
[英文文案]

#### 🇪🇸 Español
[西班牙语文案]

#### 🇯🇵 日本語
[日语文案]

---

翻译完成！你可以：
- 回复「调整」来修改某个语言版本
- 回复「保存」将所有内容保存到输出目录
- 或者继续分析新的文案
```

## Global Rules

### 对话规范
- **简洁高效**：每个阶段完成后简短告知，不要冗长解释
- **自动推进**：分析完成后自动进入下一阶段，除非用户要求暂停
- **保持上下文**：记住本次会话中分析过的所有文案和生成的变体

### 内容质量标准
- **合规性**：避免敏感词、夸大宣传、违规承诺
- **风格要求**：自然亲切，像朋友分享
  - ❌ 避免："限时抢购"、"错过后悔"、"史上最低"
  - ✅ 推荐："我最近发现"、"真的好用"、"分享给你们"
- **真实性**：不编造数据和效果，保持真诚

### 样本库管理
- **自动保存**：每次分析后自动保存到 `samples/cases/`
- **索引更新**：同步更新 `samples/index.json`
- **学习机制**：随着样本增多，生成质量会提升
- **隐私保护**：如果文案中包含品牌名/产品名，在样本中用占位符替换

### 文件操作
- **路径处理**（Windows）：使用 Git Bash，路径用双引号，正斜杠
- **输出目录**：生成的文案保存到 `output/` 目录
- **命名规范**：`YYYYMMDD-HHMMSS-变体标签.md`

### 迭代优化
- 用户可以随时说"重新生成"、"换个角度"、"更口语化"等要求
- 每次迭代都会参考之前的反馈，持续优化

## Advanced Features

### 批量处理
如果用户一次性提供多段文案，询问处理方式：
- 逐个分析并生成变体
- 或者先全部分析，再统一生成

### 对比模式
如果用户提供 2 段文案，自动进入对比模式：
- 分析两段文案的差异
- 指出各自的优势和劣势
- 生成融合两者优点的新变体

### 学习反馈
用户可以告诉 AI 哪些变体效果好（如"变体2转化率很高"），系统会记录并在未来生成时参考。

## Error Handling

- 如果用户输入的内容太短（< 20 字），提示："这段文案有点短，能提供更完整的内容吗？"
- 如果内容不像文案（如代码、数据），询问："这看起来不像营销文案，你是想让我做什么呢？"
- 如果翻译请求不明确，列出可选语言让用户选择
