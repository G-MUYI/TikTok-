# Project Guidelines

## gstack

Use the /browse skill from gstack for all web browsing. Never use mcp__claude-in-chrome__* tools.

Available skills:
- /office-hours - YC Office Hours style product consultation
- /plan-ceo-review - CEO-level product review
- /plan-eng-review - Engineering architecture review
- /plan-design-review - Design review and audit
- /design-consultation - Complete design system creation
- /review - Staff engineer code review
- /ship - Release engineering and PR creation
- /land-and-deploy - Merge and deploy to production
- /canary - Post-deploy monitoring
- /benchmark - Performance benchmarking
- /browse - Real browser automation
- /qa - QA testing with bug fixes
- /qa-only - QA testing report only
- /design-review - Design audit with fixes
- /setup-browser-cookies - Import browser cookies
- /setup-deploy - Configure deployment
- /retro - Weekly retrospective
- /investigate - Systematic debugging
- /document-release - Update project documentation
- /codex - OpenAI Codex second opinion
- /cso - Security audit (OWASP + STRIDE)
- /autoplan - Automated review pipeline
- /careful - Safety guardrails
- /freeze - Edit lock
- /guard - Full safety mode
- /unfreeze - Remove edit lock
- /gstack-upgrade - Upgrade gstack

If gstack skills aren't working, run: `cd .claude/skills/gstack && ./setup`
