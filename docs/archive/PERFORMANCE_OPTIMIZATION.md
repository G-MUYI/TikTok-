# 性能优化与挡位控制说明

> 更新日期：2026-03-05
> 版本：v1.3.0

## 优化目标

1. **提供分析挡位选择**：用户可根据需求选择快速/标准/深度三档分析模式
2. **加速默认流程**：默认快速模式，将分析时间从 20-30秒 缩短至 **10秒内**
3. **日语时长约束**：确保日语版本在 **20秒口播时长**内完成，同时保持爆款结构完整性

---

## 一、分析挡位系统 ⚙️

### 设计理念
不同场景对分析深度的需求不同：
- **快速生成场景**：追求效率，快速获得可用文案
- **学习研究场景**：深入理解爆款原理，需要全维度分析
- **日常使用场景**：平衡深度与速度

### 三档模式对比

| 挡位 | 时长 | 维度数 | 适用场景 | 触发指令 |
|------|------|--------|---------|---------|
| 🚀 快速 | 10秒 | 3个核心维度 | 快速生成文案，追求效率 | `/爆款文案` 或 `--快速` |
| ⚡ 标准 | 15秒 | 4个维度 | 日常使用，平衡深度与速度 | `--标准` |
| 🔬 深度 | 20-30秒 | 5个完整维度 | 学习爆款原理，深入研究 | `--深度` |

### 各挡位详细说明

#### 🚀 快速模式（默认）

**分析维度**：
1. 爆款公式（结构概括）
2. 核心要素（情绪/视角/心理触发/人群）
3. 可复用模式（1-2个关键技巧）

**输出示例**：
```markdown
📊 **文案核心分析**（快速版）

### 🎯 爆款公式
**[反问钩子] + [痛点共鸣] + [解决方案] + [行动暗示]**

### 💡 核心要素
- **情绪基调**：好奇 + 惊喜（强度8分）
- **叙事视角**：第一人称朋友分享
- **心理触发**：社会认同、损失厌恶
- **目标人群**：25-40岁女性，关注生活品质

### 🔑 可复用模式
反问开头制造悬念 + 具体数字增强可信度
```

**优势**：
- ✅ 10秒内完成，效率最高
- ✅ 提取核心要素，不丢失关键信息
- ✅ 适合批量处理和快速迭代

#### ⚡ 标准模式

**分析维度**：
1. 爆款公式（结构概括）
2. 核心要素（情绪/视角/语言特点/心理触发/人群）
3. 可复用模式（2-3个关键技巧）

**新增内容**：
- 情绪转折点分析
- 语言特点（口语化程度/修辞手法）
- 2-3个心理学原理（而非快速模式的1-2个）

**优势**：
- ✅ 15秒完成，速度与深度平衡
- ✅ 提供更多语言层面的洞察
- ✅ 适合日常使用

#### 🔬 深度模式

**分析维度**：
1. 结构分析（开头/中间/结尾/节奏）
2. 情绪分析（主导情绪/强度/转折点）
3. 语言特点（人称/语气/修辞/关键词密度）
4. 认知劫持与心理学原理（7个维度）
5. 目标受众画像（年龄/痛点/期望）

**优势**：
- ✅ 全维度分析，深入理解爆款原理
- ✅ 适合学习和研究
- ✅ 可作为培训教材

**劣势**：
- ⚠️ 耗时较长（20-30秒）
- ⚠️ 输出内容较多（500+字）

### 使用建议

| 场景 | 推荐挡位 | 理由 |
|------|---------|------|
| 日常生成文案 | 快速 | 效率优先，快速获得结果 |
| 重要项目文案 | 标准 | 平衡深度，确保质量 |
| 学习爆款原理 | 深度 | 全面理解，深入学习 |
| 批量处理 | 快速 | 节省时间，提高产能 |
| 首次使用 | 标准 | 体验完整功能 |

## 二、日语时长约束 ⏱

### 需求背景
日本 TikTok 用户习惯更短的视频节奏，20秒是黄金时长。超过此时长会导致：
- 完播率下降
- 用户流失
- 算法推荐权重降低

### 技术实现

#### 1. 时长计算公式
```
日语口播时长 = モーラ数 ÷ 口播速度
标准口播速度 = 6-7.5モーラ/秒
20秒目标 = 130-150モーラ
```

**モーラ（mora）说明**：
- 日语的音节单位
- 例：「ありがとう」= 5モーラ（a-ri-ga-to-u）
- 例：「きょう」= 2モーラ（kyo-u）

#### 2. 优化策略

##### 策略一：口语缩略
| 标准形 | 口语缩略形 | 节省モーラ |
|--------|-----------|-----------|
| たまらない | たまんない | 1 |
| わからない | わかんない | 1 |
| ～ている | ～てる | 1 |
| ～ておく | ～とく | 1 |

##### 策略二：删除冗余修饰
```markdown
# 优化前（180モーラ，约24秒）
何年もお花を育ててきたけど、正直失敗ばっかりだった。根腐れ、日焼け、霜でダメになって…

# 优化后（140モーラ，约19秒）
もうね、何年もお花枯らしてきたんだけど、睡蓮だけはマジで別。
```

##### 策略三：短句替代长句
```markdown
# 优化前（长句，信息密度低）
この子のすごいところは、本当に手がかからないことです。

# 优化后（短句，信息密度高）
お水張るだけでいいの。
```

#### 3. 结构完整性验证
即使精简到20秒，也必须保留爆款文案的4个核心部分：

| 核心部分 | 时长分配 | 示例 |
|---------|---------|------|
| 开头钩子 | 3-4秒 | もうね、何年もお花枯らしてきたんだけど |
| 价值展示 | 6-8秒 | お水張るだけでいいの。咲いたら部屋中いい香り |
| 情绪共鸣 | 4-5秒 | 凍っても春に咲くし、半年以上もつの |
| 行动暗示 | 3-4秒 | 数百円でこれ、ほんとコスパやばい |

#### 4. 输出格式
生成日语版本后，必须包含：
```markdown
#### 🇯🇵 日本語（20秒版）⏱
[日语文案内容]

**⏱ 时长验证**：约 18 秒（基于 7モーラ/秒）

**🎯 结构保留检查**：
- ✅ 开头钩子：保留（もうね、何年も...）
- ✅ 价值展示：保留（お水張るだけ...）
- ✅ 情绪共鸣：保留（凍っても春に...）
- ✅ 行动暗示：保留（数百円でこれ...）
```

## 五、注意事项

### 1. 挡位选择建议
- **默认快速模式**：适合90%的日常使用场景
- **标准模式**：适合重要项目或需要更多洞察时
- **深度模式**：仅用于学习研究，不建议日常使用

### 2. 分析质量保证
- 快速模式精简的是描述，不是洞察
- 关键信息保留率 100%
- 可复用模式依然会被提取

### 3. 日语时长约束的边界
- 目标：20秒（130-150モーラ）
- 可接受范围：18-22秒
- 如果内容复杂度高，可适当放宽至 25秒，但需标注说明

### 4. 结构完整性优先
- 宁可时长略超，也不能破坏爆款结构
- 如果无法在20秒内保留所有4个核心部分，优先保留：
  1. 开头钩子（必须）
  2. 价值展示（必须）
  3. 情绪共鸣（可精简）
  4. 行动暗示（可隐含）

### 5. 用户偏好记忆
- 系统会记录用户选择的挡位偏好
- 保存在 `.ai/user-profile.json`
- 后续会话自动应用偏好设置
