# 会话持久化使用指南

## 🎯 功能说明

会话持久化功能让AI能够记住你的工作状态和偏好，即使关闭窗口后重新打开，也能继续之前的工作。

## 📁 状态文件说明

### 1. `.ai/session-state.json` - 当前会话状态
**由AI自动维护**

保存内容：
- 当前正在进行的任务
- 最后分析的文案
- 已生成的变体
- 待翻译的内容
- 用户反馈

**何时更新**：
- 每次完成文案分析
- 每次生成变体
- 每次收到用户反馈

### 2. `.ai/user-profile.json` - 用户偏好配置
**可手动编辑**

保存内容：
- 默认参数设置（变体数量、风格等）
- 喜欢的文案模式
- 避免使用的词汇
- 历史反馈数据

**何时更新**：
- 用户提供反馈时
- 用户修改配置时
- AI学习到新的偏好时

### 3. `.ai/work-history.json` - 工作历史记录
**由AI自动维护**

保存内容：
- 最近的会话记录
- 最近分析的文案列表
- 最近生成的变体列表
- 使用统计数据

**何时更新**：
- 每次会话结束时
- 定期汇总统计数据

## 🚀 使用方法

### 自动恢复（推荐）

AI会在启动时自动检查状态文件：

```
AI: 👋 欢迎回来！

我注意到上次你分析了一段美妆类文案，已生成3个变体但还未翻译。

你想：
1. 继续翻译上次的变体
2. 重新开始新的分析
3. 查看上次的分析结果

请告诉我你的选择。
```

### 手动恢复

如果AI没有自动提示，你可以使用指令：

```
/恢复会话
```

AI会读取状态文件并显示上次的工作内容。

### 查看历史

查看最近的工作记录：

```
/查看历史
```

AI会显示：
- 最近分析的5段文案
- 最近生成的5个变体
- 最近保存的5个输出文件

### 清空状态

如果想重新开始，清空所有状态：

```
/清空状态
```

AI会清空 `session-state.json`，但保留 `user-profile.json` 中的用户偏好。

## ⚙️ 自定义偏好

你可以手动编辑 `.ai/user-profile.json` 来自定义偏好：

### 示例：修改默认参数

```json
{
  "preferences": {
    "default_variant_count": 5,        // 默认生成5个变体
    "default_style": "professional",   // 默认专业风格
    "auto_translate": true             // 自动翻译
  }
}
```

### 示例：添加避免词汇

```json
{
  "style_preferences": {
    "avoid_words": [
      "限时抢购",
      "错过后悔",
      "史上最低",
      "你的自定义词汇"
    ]
  }
}
```

### 示例：设置喜欢的模式

```json
{
  "style_preferences": {
    "favorite_patterns": [
      "好奇心驱动",
      "故事叙述",
      "你喜欢的模式"
    ]
  }
}
```

## 🔄 工作流程示例

### 场景1：中断后继续

**第一次会话**：
```
你: /爆款文案
[粘贴文案]

AI: [完成分析和生成变体]

你: [关闭窗口]
```

**第二次会话**：
```
AI: 👋 欢迎回来！上次你生成了3个变体，需要翻译吗？

你: 是的，翻译变体2

AI: [提供翻译]
```

### 场景2：学习用户偏好

**多次使用后**：
```
你: /爆款文案
[粘贴文案]

AI: 根据你的偏好，我会：
- 生成5个变体（你通常喜欢更多选择）
- 使用"故事叙述"风格（你之前多次选择这种风格）
- 避免使用"限时抢购"等词汇

正在分析中...
```

## 🛡️ 隐私说明

- **本地存储**：所有状态文件仅保存在本地，不会上传
- **内容引用**：状态文件不保存完整的文案内容，仅保存引用和摘要
- **随时删除**：你可以随时删除状态文件，不影响系统功能
- **手动控制**：你可以使用 `/清空状态` 清空会话数据

## 📊 统计功能

查看使用统计：

```
/统计
```

AI会显示：
- 总共分析了多少段文案
- 生成了多少个变体
- 最常使用的功能
- 最喜欢的文案模式

## ❓ 常见问题

### Q: 状态文件会占用多少空间？
A: 非常小，通常每个文件不超过10KB。

### Q: 如果我不想保存状态怎么办？
A: 可以在每次使用时加上 `--跳过保存` 参数，或者删除状态文件。

### Q: 状态文件会过期吗？
A: 不会自动过期，但你可以手动清空或删除。

### Q: 多个项目如何管理？
A: 每个项目有独立的 `.ai/` 目录，状态互不影响。

### Q: 如何备份我的偏好设置？
A: 复制 `.ai/user-profile.json` 文件即可。

## 🔧 故障排除

### 问题：AI没有自动恢复会话
**解决方案**：
1. 检查 `.ai/session-state.json` 是否存在
2. 手动使用 `/恢复会话` 指令
3. 如果文件损坏，删除后重新开始

### 问题：偏好设置没有生效
**解决方案**：
1. 检查 `.ai/user-profile.json` 格式是否正确（JSON格式）
2. 确认修改后保存了文件
3. 重新启动AI会话

### 问题：想重置所有设置
**解决方案**：
```bash
# 删除所有状态文件
rm .ai/session-state.json
rm .ai/user-profile.json
rm .ai/work-history.json

# 或使用指令
/清空状态 --全部
```

---

**提示**：会话持久化功能会随着使用越来越智能，AI会学习你的偏好并自动优化生成策略。
