# 会话持久化功能测试指南

## 🎯 测试目标

验证会话持久化功能是否正常工作，包括：
- 状态保存
- 状态恢复
- 用户偏好学习
- 工作历史记录

## 📋 测试前准备

### 1. 检查状态文件是否存在

```bash
cd "d:/AI导航网站及资源网站/AI编程小工具/TikTok-爆款文案生成器/TikTok-"
ls -la .ai/*.json
```

应该看到：
- `.ai/session-state.json`
- `.ai/user-profile.json`
- `.ai/work-history.json`

### 2. 查看初始状态

```bash
cat .ai/session-state.json
cat .ai/user-profile.json
```

## 🧪 测试用例

### 测试1：基础状态保存

**目的**：验证AI能否保存会话状态

**步骤**：
1. 在新的对话窗口中输入：
   ```
   /爆款文案

   姐妹们！我发现了一个宝藏面膜，用了三天皮肤真的变嫩了！
   以前我也不信这些，但这次真的不一样。
   早上起来照镜子，我自己都吓一跳，皮肤居然这么透亮！
   ```

2. 等待AI完成分析和生成变体

3. **不要继续翻译**，直接关闭对话窗口

4. 检查状态文件是否更新：
   ```bash
   cat .ai/session-state.json
   ```

**期望结果**：
- `session-state.json` 中应该有：
  - `current_session` 不为空
  - `context.last_analyzed_copy` 包含文案摘要
  - `context.generated_variants` 包含生成的变体
  - `context.pending_translation` 为 true

### 测试2：自动恢复会话

**目的**：验证AI能否自动恢复上次会话

**步骤**：
1. 打开新的对话窗口

2. 观察AI的第一句话

**期望结果**：
AI应该主动提示：
```
👋 欢迎回来！

我注意到上次你分析了一段美妆类文案，已生成3个变体但还未翻译。

你想：
1. 继续翻译上次的变体
2. 重新开始新的分析
3. 查看上次的分析结果
```

### 测试3：手动恢复会话

**目的**：验证手动恢复指令是否有效

**步骤**：
1. 如果AI没有自动提示，输入：
   ```
   /恢复会话
   ```

**期望结果**：
AI应该显示上次的工作内容，并询问是否继续。

### 测试4：用户偏好学习

**目的**：验证AI能否学习用户偏好

**步骤**：
1. 完成一次完整的文案生成流程

2. 当AI询问反馈时，回复：
   ```
   我最喜欢变体2，因为更口语化
   ```

3. 检查用户配置文件：
   ```bash
   cat .ai/user-profile.json
   ```

4. 再次使用 `/爆款文案` 分析新文案

**期望结果**：
- `user-profile.json` 中应该记录了偏好
- AI在生成新变体时应该提到会参考之前的偏好

### 测试5：查看历史

**目的**：验证历史记录功能

**步骤**：
1. 输入：
   ```
   /查看历史
   ```

**期望结果**：
AI应该显示：
- 最近分析的文案列表
- 最近生成的变体列表
- 使用统计数据

### 测试6：清空状态

**目的**：验证清空状态功能

**步骤**：
1. 输入：
   ```
   /清空状态
   ```

2. 检查状态文件：
   ```bash
   cat .ai/session-state.json
   ```

**期望结果**：
- `session-state.json` 中的 `current_session` 应该为空
- `user-profile.json` 应该保持不变（用户偏好不清空）

### 测试7：跨会话持久化

**目的**：验证多次会话的持久化

**步骤**：
1. 第一次会话：分析文案A，生成变体
2. 关闭窗口
3. 第二次会话：继续翻译
4. 关闭窗口
5. 第三次会话：分析文案B
6. 输入 `/查看历史`

**期望结果**：
- 历史记录中应该包含文案A和文案B
- 统计数据应该累积

## 🐛 常见问题排查

### 问题1：AI没有自动恢复会话

**可能原因**：
- 状态文件格式错误
- 状态文件为空
- AI没有读取状态文件

**排查步骤**：
1. 检查文件是否存在：`ls -la .ai/*.json`
2. 检查文件内容：`cat .ai/session-state.json`
3. 检查JSON格式：`cat .ai/session-state.json | python -m json.tool`
4. 手动触发恢复：`/恢复会话`

### 问题2：状态文件没有更新

**可能原因**：
- AI没有执行保存操作
- 文件权限问题

**排查步骤**：
1. 检查文件修改时间：`ls -l .ai/*.json`
2. 检查文件权限：`ls -la .ai/*.json`
3. 手动编辑文件测试：`echo '{}' > .ai/test.json`

### 问题3：用户偏好没有生效

**可能原因**：
- 偏好格式错误
- AI没有读取偏好文件

**排查步骤**：
1. 检查 `user-profile.json` 内容
2. 验证JSON格式
3. 重新启动会话
