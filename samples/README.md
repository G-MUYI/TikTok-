# 样本库管理指南

## 样本库的作用

样本库是这个系统的"大脑"，它会：
1. 保存每次分析的爆款文案
2. 提取可复用的模式和技巧
3. 随着样本增多，生成质量越来越高
4. 形成你的专属文案知识库

## 目录结构

```
samples/
├── index.json          # 样本索引（自动维护）
├── template.md         # 样本格式模板
└── cases/              # 具体案例
    ├── 001-xxx.md
    ├── 002-xxx.md
    └── ...
```

## 样本索引格式（index.json）

```json
{
  "version": "1.0.0",
  "last_updated": "2026-03-04",
  "total_samples": 3,
  "samples": [
    {
      "id": "001",
      "title": "护肤品种草文案",
      "file": "cases/001-skincare-viral.md",
      "tags": ["护肤", "好奇心驱动", "社会认同"],
      "product_type": "美妆护肤",
      "emotion": "兴奋",
      "added_date": "2026-03-04",
      "performance": {
        "likes": 50000,
        "comments": 1200,
        "shares": 800
      }
    }
  ]
}
```

## 样本文件格式（cases/*.md）

每个样本文件包含：
- 原始文案
- 深度分析（结构/情绪/心理学原理）
- 关键洞察
- 标签和元数据

详见 `template.md`

## 自动化流程

### 1. 自动保存
每次你提供文案进行分析时，AI 会：
1. 生成唯一 ID（递增编号）
2. 创建样本文件到 `cases/`
3. 更新 `index.json`

### 2. 自动学习
AI 会从样本库中学习：
- 哪些开头方式最有效
- 哪些情绪基调转化率高
- 哪些心理学原理最常用
- 你的个人偏好和风格

### 3. 隐私保护
如果文案中包含：
- 具体品牌名 → 替换为 `[品牌名]`
- 产品名称 → 替换为 `[产品]`
- 价格信息 → 替换为 `[价格]`

## 手动管理

### 查看所有样本
```bash
cat samples/index.json
```

### 查看特定样本
```bash
cat samples/cases/001-xxx.md
```

### 搜索样本
按标签搜索：
```bash
grep -r "#好奇心驱动" samples/cases/
```

按产品类型搜索：
```bash
grep "产品类型：美妆护肤" samples/cases/*.md
```

### 删除样本
如果某个样本不再需要：
1. 删除 `cases/` 中的文件
2. 手动更新 `index.json`（或让 AI 重新生成索引）

## 样本质量标准

### 好的样本应该包含：
- ✅ 完整的原始文案（至少 50 字）
- ✅ 清晰的结构分析
- ✅ 深入的心理学原理解读
- ✅ 可复用的关键洞察
- ✅ 准确的标签和分类

### 避免保存：
- ❌ 过于简短的文案（< 20 字）
- ❌ 不符合平台规范的内容
- ❌ 纯粹的产品描述（没有营销技巧）

## 样本库优化建议

### 1. 定期回顾
每月回顾一次样本库：
- 哪些模式最有效？
- 哪些样本可以删除？
- 需要补充哪些类型的样本？

### 2. 分类管理
可以按产品类型分类：
- 美妆护肤
- 食品饮料
- 数码电子
- 服装配饰
- 家居生活

### 3. 标签体系
建议使用的标签：
- **情绪类**：#兴奋 #好奇 #共鸣 #焦虑
- **技巧类**：#故事叙述 #数据背书 #对比反差
- **原理类**：#稀缺性 #社会认同 #损失厌恶
- **风格类**：#朋友分享 #专家建议 #用户见证

### 4. 数据追踪
如果有数据，记录下来：
- 点赞数
- 评论数
- 转化率
- 播放量

这些数据会帮助 AI 学习什么样的文案更有效。

## 导出和备份

### 导出样本库
```bash
zip -r samples-backup-20260304.zip samples/
```

### 导入样本库
如果你有其他项目的样本库，可以合并：
1. 复制 `cases/` 文件到当前项目
2. 让 AI 重新生成 `index.json`
